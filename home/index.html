🎯 What Is This Challenge?

You'll race through 4 enterprise software challenges using GitHub Copilot as your AI pair programmer. Each challenge focuses on a critical pillar of production-ready software: documentation, custom instructions, custom agents, and testing.

Your mission is to apply Copilot's AI capabilities to transform the Snake Cash Rush game project into enterprise-grade code — one challenge at a time.

📋 Pre-requisites

  • Git — Download Git
  • Visual Studio Code — Download VS Code
  • A modern browser (Edge or Chrome)

  • GitHub Copilot Chat — Ask questions, generate code, and get explanations
  • A modern browser — Microsoft Edge or Chrome to run the game

⚙️ Setting Up GitHub Copilot in VS Code

  1. Open VS Code and go to the Extensions panel (Ctrl+Shift+X)
  2. Search for "GitHub Copilot" and install the extension
  3. Also install "GitHub Copilot Chat" for the chat panel
  4. Sign in with your GitHub account when prompted
  5. Verify Copilot is active — look for the Copilot icon in the status bar
  6. Open Copilot Chat with Ctrl+Shift+I or click the chat icon in the sidebar

🤖 Custom Agent — Spec-Driven Development

TURBO

Build a custom Copilot agent that drives Spec-Driven Development (SDD). This agent guides users through a structured 3-phase workflow to produce comprehensive spec documents before any code is written.

🎯 Your Mission

Create a custom Copilot Chat agent (using a .agent.md file) that runs a 3-phase spec-driven development workflow. Given a user prompt, the agent produces spec documents through phases: Intent, Design, and Tasks + Executive Summary.

🏢 Enterprise Value

Spec-Driven Development ensures alignment between stakeholders and engineers before code is written — reducing rework, miscommunication, and scope creep. A custom agent that automates spec generation encodes your organization's planning standards into a repeatable process, accelerating delivery while maintaining governance and traceability.

📝 Steps to Complete

  1. Stay on your branch — continue on the same branch you created in Challenge 1
  2. Create a .github/agents/spec-driven-dev.agent.md file defining the custom agent
  3. Define 3 phases the agent must execute sequentially:
    • Phase 1 — Intent: Capture the user's goal, constraints, and acceptance criteria
    • Phase 2 — Design: Produce a technical design covering architecture, components, and data flow
    • Phase 3 — Tasks & Summary: Break the design into actionable tasks and produce an executive summary
  4. Test the agent — invoke it in Copilot Chat with a feature request and verify it produces all spec documents
  5. Save the generated specs as markdown files in a specs/ folder
  6. Commit and push your changes to the branch

💡 Sample Prompts

Use this prompt in Copilot Chat to create your spec-driven development agent:

Create a custom GitHub Copilot agent file at .github/agents/spec-driven-dev.agent.md. This agent is a Spec-Driven Development assistant that takes a user's feature request and runs through the following phases sequentially. The agent should save all outputs as markdown files in a specs/[feature-name]/ folder. Each phase must build on the previous phase's output. The agent should confirm with the user before proceeding to the next phase: Phase 1 — Intent: Ask clarifying questions, then produce an intent.md capturing the goal, problem statement, target users, constraints, and acceptance criteria. Phase 2 — Design: Based on the intent, produce a design.md with the technical architecture, component breakdown, data flow, API contracts, and key design decisions. Phase 3 — Tasks & Summary: Break the design into an ordered task list in tasks.md with clear deliverables per task, then produce a summary.md executive summary suitable for stakeholders covering scope, approach, risks, and estimated effort. NB Ensure you prompt the user if they want to make changes after each phase and approve it. 📋 Click to copy

After creating the agent, select it from the agent selection dropdown at the top of the Copilot Chat panel (click the model/agent picker and choose spec-driven-dev). Then test your agent with a prompt like this:

I want to add a power-up system to the Snake Cash Rush game where the snake can collect special items that give temporary abilities like speed boost, invincibility, or double points 📋 Click to copy

🧪 Testing

FINISH LINE

Enterprise software demands rigorous testing. Use Copilot to build a comprehensive test suite that validates your game logic and catches regressions before they reach production.

🎯 Your Mission

Generate a comprehensive test suite with unit tests covering happy paths, edge cases, and failure scenarios for the Snake Cash Rush game.

🏢 Enterprise Value

Automated tests are the safety net that enables rapid iteration without fear. In enterprise environments, comprehensive test coverage reduces production incidents, enables confident refactoring, supports CI/CD pipelines, and provides living documentation of expected behavior — ultimately protecting revenue and customer trust.

📝 Steps to Complete

  1. Stay on your branch — continue on the same branch you created in Challenge 1
  2. Open GitHub Copilot Chat in VS Code (Ctrl+Shift+I)
  3. Generate unit tests for the core game logic (movement, collision, scoring)
  4. Add edge case tests — boundary conditions, invalid inputs, state transitions
  5. Run the tests to verify they pass against the current implementation
  6. Commit and push your changes to the branch
